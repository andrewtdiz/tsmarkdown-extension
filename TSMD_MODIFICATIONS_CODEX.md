# tmLanguage vs Spec Discrepancies

- TSM blocks are only recognized after `return (` or `=> (` in the grammar (`syntaxes/tsmd.tmLanguage.json:123-155`), but the spec allows them in any expression position (`TSMD_FILE_SPEC.md:4-10`, `TSMD_FILE_SPEC.md:58-60`). Assignments, conditional branches, or ternaries using TSM blocks will miss the `meta.tsm-block.tsmd` scope.
- Nested interpolation blocks only trigger when a ternary `? (` appears (`syntaxes/tsmd.tmLanguage.json:192-220`), yet the spec lists `&& (...)` and false-branch `: (...)` idioms as supported (`TSMD_FILE_SPEC.md:22-30`). Those forms do not receive markdown/component scopes inside the interpolation today.
- Component closing tags are described as needing an exact name match (`TSMD_FILE_SPEC.md:35-39`), but the `end` pattern for `<@...>` is malformed (`syntaxes/tsmd.tmLanguage.json:345-367`), missing the final parenthesis and any back-reference. As written it fails to match `</@Name>` reliably, so closing tags never enter `entity.name.tag.component.tsmd`.
- Wrapper tags must not be self-closing per the spec (`TSMD_FILE_SPEC.md:45-49`), yet `tsm-xml-group` accepts a `/>` terminator (`syntaxes/tsmd.tmLanguage.json:428-470`), which lets `<content />` highlight even though the construct is invalid.
- Markdown inside blocks should preserve indentation (see `TSMD_FILE_SPEC.md:12-15` and the indented example at `TSMD_FILE_SPEC.md:66-74`), but the `tsm-markdown` heading patterns require the hash to sit at column 0 (`syntaxes/tsmd.tmLanguage.json:479-500`). Indented headings such as `  # Title` therefore fall back to plain text highlighting.
